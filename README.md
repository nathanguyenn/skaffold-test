# microservice-example

A small but real **.NET 10 Minimal API** backed by **PostgreSQL** (via EF Core 10 + Npgsql).
It manages `projects` and their `tasks` (one-to-many), with a genuine relational schema,
EF Core migrations, liveness/readiness health checks, unit + integration tests, and a
distroless (chiseled) container image.

It is designed to be container-deployed to Kubernetes, but runs locally against any Postgres.

---

## Architecture

```
                 ┌───────────────────────────────────────────┐
   HTTP :8080    │  ASP.NET Core 10 Minimal API               │
 ───────────────▶  Program.cs                                 │
                 │   ├─ /healthz  (liveness  — no dependencies)│
                 │   ├─ /readyz   (readiness — DB connectivity)│
                 │   └─ /projects … (CRUD, ProjectEndpoints)   │
                 │            │                                │
                 │            ▼                                │
                 │  AppDbContext (EF Core 10)                  │
                 └────────────┼────────────────────────────────┘
                              │ Npgsql
                              ▼
                 ┌───────────────────────────┐
                 │ PostgreSQL                 │
                 │   projects ──1:N──▶ tasks  │
                 └───────────────────────────┘
```

### Tech stack
| Concern | Choice |
|---|---|
| Runtime / TFM | .NET 10 (`net10.0`) |
| Web | ASP.NET Core Minimal API |
| Data | EF Core 10 + `Npgsql.EntityFrameworkCore.PostgreSQL` 10.0.2 |
| Health | `Microsoft.Extensions.Diagnostics.HealthChecks.EntityFrameworkCore` (`AddDbContextCheck`) |
| Tests | xUnit + `Microsoft.AspNetCore.Mvc.Testing` + EF Core InMemory |
| Container | `mcr.microsoft.com/dotnet/aspnet:10.0-noble-chiseled` (distroless, non-root) |
| Migrations tool | `dotnet-ef` 10.0.9 (local tool, see `dotnet-tools.json`) |

### Data model
```
projects                          tasks
  id          bigint PK (identity)  id          bigint PK (identity)
  name        text   NOT NULL       project_id  bigint NOT NULL  FK → projects(id) ON DELETE CASCADE
  created_at  timestamptz now()     title       text   NOT NULL
                                    is_done     boolean NOT NULL default false
                                    created_at  timestamptz now()
                                    INDEX ix_tasks_project_id (project_id)
```

### HTTP endpoints
| Method | Route | Purpose |
|---|---|---|
| GET | `/healthz` | Liveness — process is up (runs **no** checks, never touches the DB) |
| GET | `/readyz` | Readiness — opens a DB connection; 200 healthy / 503 if DB unreachable |
| GET | `/projects` | List projects |
| POST | `/projects` | Create project — body `{ "name": "..." }` |
| GET | `/projects/{id}/tasks` | List a project's tasks (404 if project missing) |
| POST | `/projects/{id}/tasks` | Create a task — body `{ "title": "..." }` |
| GET | `/swagger` | Interactive **Swagger UI** |
| GET | `/openapi/v1.json` | OpenAPI 3.1 document (generated by `Microsoft.AspNetCore.OpenApi`, consumed by Swagger UI) |

## Configuration

The only required setting is the connection string, read from
**`ConnectionStrings__Default`** (ASP.NET Core's double-underscore env convention):

```
Host=<host>;Port=5432;Database=dotnet_lab;Username=dotnet_lab;Password=<pw>
```

## Run it locally

Prerequisites: **.NET 10 SDK** (`dotnet --version` → `10.0.3xx`) and **Docker** (for the
throwaway Postgres below). Run all commands from the repo root.

### 1. Start a local Postgres (matches the cluster's PG 18)
```bash
docker run -d --name mse-pg \
  -e POSTGRES_USER=dotnet_lab -e POSTGRES_PASSWORD=devpw -e POSTGRES_DB=dotnet_lab \
  -p 5433:5432 postgres:18
```

### 2. Point the app at it, apply migrations, run
```bash
export ConnectionStrings__Default="Host=localhost;Port=5433;Database=dotnet_lab;Username=dotnet_lab;Password=devpw"

dotnet tool restore                              # restores dotnet-ef
dotnet ef database update --project src/MicroserviceExample   # creates the tables
dotnet run --project src/MicroserviceExample     # listens on http://localhost:5xxx (see console)
```

### 3. Exercise it
```bash
BASE=http://localhost:5xxx   # use the port printed by `dotnet run`

curl -s $BASE/healthz                                            # Healthy
curl -s $BASE/readyz                                             # Healthy (DB reachable)
curl -s -XPOST $BASE/projects -H 'content-type: application/json' -d '{"name":"Apollo"}'
curl -s $BASE/projects
curl -s -XPOST $BASE/projects/1/tasks -H 'content-type: application/json' -d '{"title":"Land on the moon"}'
curl -s $BASE/projects/1/tasks
```

### Cleanup
```bash
docker rm -f mse-pg
```

### Alternative: run against an existing Postgres
Point `ConnectionStrings__Default` at any reachable Postgres instead of the local container
(e.g. a managed/remote database reached over a port-forward or tunnel):
```bash
export ConnectionStrings__Default="Host=<host>;Port=5432;Database=<db>;Username=<user>;Password=<pw>"
dotnet run --project src/MicroserviceExample
```
> Note: for a Kubernetes deployment, apply schema migrations with a dedicated EF
> migrations-bundle Job rather than at app startup. Only run `dotnet ef database update`
> by hand if you intend to apply the schema yourself.

## Tests
```bash
dotnet test
```
Covers the endpoint handlers (validation, CRUD, 404s) against an in-memory `AppDbContext`,
plus a `WebApplicationFactory` integration test that boots the real app and asserts `/healthz`.

## Build & run the container
```bash
docker build -t microservice-example:dev .
docker run --rm -p 8080:8080 \
  -e ConnectionStrings__Default="Host=host.docker.internal;Port=5433;Database=dotnet_lab;Username=dotnet_lab;Password=devpw" \
  microservice-example:dev
# → http://localhost:8080/healthz
```

## Migrations
```bash
dotnet ef migrations add <Name> --project src/MicroserviceExample   # add a migration
dotnet ef database update      --project src/MicroserviceExample   # apply to the configured DB
dotnet ef migrations bundle    --project src/MicroserviceExample   # self-contained executable for CI/k8s Job
```
